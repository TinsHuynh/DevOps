SonarQube on Ubuntu 22.04 LTS 

SonarQube là một công cụ mã nguồn mở để phân tích chất lượng mã. Công cụ này có thể quét mã nguồn để tìm lỗi và lỗ hổng tiềm ẩn và tạo báo cáo cho phép bạn xác định các vấn đề. Công cụ này quét tới 30 ngôn ngữ lập trình. 

SonarQube có hai phần: ứng dụng quét trên máy cục bộ để quét mã và ứng dụng máy chủ để lưu giữ hồ sơ. 

Điều kiện tiên quyết 

Triển khai máy chủ Ubuntu 22.04 với ít nhất 2GB RAM và 2 vCPU. 

Tạo người dùng không phải root với quyền sudo. 

Cập nhật máy chủ. 

Tên miền (sonarqube.example.com) 

Cấu hình Firewall 

$ sudo ufw allow http 

$ sudo ufw allow https 

Kiểm tra trạng thái 

$ sudo ufw status 

Install OpenJDK 

$ sudo apt update 

$ sudo apt install openjdk-11-jdk 

Install PostgreSQL 

$ curl https://www.postgresql.org/media/keys/ACCC4CF8.asc | gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/apt.postgresql.org.gpg >/dev/null 

Thêm PostgreSQL vào repository. 

$ sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list' 

Cập nhật lại repository 

$ sudo apt update 

Install PostgreSQL 14 

$ sudo apt install postgresql postgresql-contrib 

Kiểm tra dịch vụ 

$ sudo systemctl status postgresql 

Configure PostgreSQL 

Log in to the PostgreSQL shell. 

$ sudo -u postgres psql 

Tạo một user sonaruser role. 

postgres=# CREATE ROLE sonaruser WITH LOGIN ENCRYPTED PASSWORD 'your_password'; 

Tạo một sonarqube database. 

postgres=# CREATE DATABASE sonarqube; 

Grant all privileges on the sonarqube database to the sonaruser role. 

postgres=# GRANT ALL PRIVILEGES ON DATABASE sonarqube to sonaruser; 

Exit the shell. 

postgres=# \q 

Trở về user account. 

$ exit 

Install Sonarqube 

Tải SonarQube dùng link URL 

$ wget https://binaries.sonarsource.com/Distribution/sonarqube/sonarqube-9.6.1.59531.zip 

Cài đặt package 

$sudo apt install unzip 

Giải nén package sonarqube 

$ unzip -q sonarqube-9.6.1.59531.zip 

Chuyển  the files tới thư mục /opt/sonarqube 

$ sudo mv sonarqube-9.6.1.59531 /opt/sonarqube 

Create SonarQube User 

Tạo một system user với the group cho SonarQube. 

$ sudo adduser --system --no-create-home --group --disabled-login sonarqube 

Cấp quyền cho user Sonar được phép sử dụng thư mục /opt/sonarqube. 

$ sudo chown sonarqube:sonarqube /opt/sonarqube -R 

Configure SonarQube Server 

Mở file cấu hình sonar 

$ sudo nano /opt/sonarqube/conf/sonar.properties 

Tìm hàng có nội dung như bên dưới. 

#sonar.jdbc.username= 

#sonar.jdbc.password= 

#sonar.jdbc.url=jdbc:postgresql://localhost/sonarqube?currentSchema=my_schema 

Xóa comment (#) và thêm thông tin username và password đã tạo trước. 

sonar.jdbc.username=sonaruser 

sonar.jdbc.password=your_password 

sonar.jdbc.url=jdbc:postgresql://localhost:5432/sonarqube 

Tăng bộ nhớ ảo trên hệ thống để Elasticsearch hoạt động. Mở file sysctl.conf để chỉnh sửa. 

$ Thêm nội dung ở cuối file. 

vm.max_map_count=524288 

fs.file-max=131072 

Lưu the file nhấn Ctrl+X, và Y. 

Tạo  file /etc/security/limits.d/99-sonarqube.conf 

$ sudo nano /etc/security/limits.d/99-sonarqube.confsudo nano /etc/sysctl.conf 

Thêm nội dung 

sonarqube   -   nofile   131072 

sonarqube   -   nproc    8192 

Lưu the file nhấn Ctrl+X, và Y. 

Khởi động lại máy cập nhật thay đổi 

$ sudo reboot 

Setup Sonar Service 

Tạo file servie 

$ sudo nano /etc/systemd/system/sonarqube.service 

Thêm code vào 

[Unit] 

Description=SonarQube service 

After=syslog.target network.target 

[Service] 

Type=forking 

ExecStart=/opt/sonarqube/bin/linux-x86-64/sonar.sh start 

ExecStop=/opt/sonarqube/bin/linux-x86-64/sonar.sh stop 

User=sonarqube 

Group=sonarqube 

PermissionsStartOnly=true 

Restart=always 

StandardOutput=syslog 

LimitNOFILE=131072 

LimitNPROC=8192 

TimeoutStartSec=5 

SuccessExitStatus=143 

[Install] 

WantedBy=multi-user.target 

Khởi động SonarQube service. 

 

$ sudo systemctl start sonarqube 

Kiểm tra trạng thái service. 

$ sudo systemctl status sonarqube 

Enable service để chạy tự động khi boot. 

$ sudo systemctl enable sonarqube 

Kiểm tra xem máy chủ Sonarqube có hoạt động bình thường không. 

$ curl http://127.0.0.1:9000 

<script> 

    window.baseUrl = ''; 

    window.serverStatus = 'UP'; 

    window.instance = 'SonarQube'; 

    window.official = true; 

</script> 

Dịch vụ hoạt động bình thường 

Install Nginx 

Install dependencies. 

$ sudo apt install curl gnupg2 ca-certificates lsb-release ubuntu-keyring -y 

Import Nginx's GPG signing key. 

$ curl https://nginx.org/keys/nginx_signing.key | gpg --dearmor \ 

| sudo tee /usr/share/keyrings/nginx-archive-keyring.gpg >/dev/null 

Add a repository for Nginx's stable version. 

$ echo "deb [signed-by=/usr/share/keyrings/nginx-archive-keyring.gpg arch=amd64] \ 

http://nginx.org/packages/ubuntu `lsb_release -cs` nginx" \ 

| sudo tee /etc/apt/sources.list.d/nginx.list 

Cập nhật lại the system repository list. 

 

$ sudo apt update 

Cài đặt Nginx. 

$ sudo apt install nginx 

Khởi động the Nginx server. 

$ sudo systemctl start nginx 

Install SSL 

$ sudo snap install core 

$ sudo snap refresh core 

Install Certbot. 

$ sudo snap install --classic certbot 

Tạo một symlink cho Certbot tới /usr/bin directory. 

$ sudo ln -s /snap/bin/certbot /usr/bin/certbot 

Issue the SSL Certificate. 

$ sudo certbot certonly --nginx --agree-tos --no-eff-email --staple-ocsp --preferred-challenges http -m name@example.com -d sonarqube.example.com 

Tạo a Diffie-Hellman group certificate. 

$ sudo openssl dhparam -dsaparam -out /etc/ssl/certs/dhparam.pem 4096 

Hãy chạy thử quy trình gia hạn SSL để đảm bảo nó hoạt động. 

$ sudo certbot renew --dry-run 

Configure Nginx 

Mở file nginx.conf. 

$ sudo nano /etc/nginx/nginx.conf 

Tìm line chứa /etc/nginx/conf.d/*.conf;  

Thêm nội dung vào 

server_names_hash_bucket_size  64; 

Save the file by pressing Ctrl+X, then Y. 

 

Tạo  Sonar configuration file for Nginx. 

$ sudo nano /etc/nginx/conf.d/sonar.conf 

Thêm nội dung vào. 

# Redirect HTTP to HTTPS 

server { 

    listen 80 default_server; 

    server_name sonarqube.example.com; 

    return 301 https://$server_name$request_uri; 

} 

server { 

    listen 443 ssl http2; 

    server_name sonarqube.example.com; 

    http2_push_preload on; # Enable HTTP/2 Server Push 

    ssl_certificate /etc/letsencrypt/live/sonarqube.example.com/fullchain.pem; 

    ssl_certificate_key /etc/letsencrypt/live/sonarqube.example.com/privkey.pem; 

    ssl_trusted_certificate /etc/letsencrypt/live/sonarqube.example.com/chain.pem; 

    ssl_session_timeout 1d; 

    ssl_protocols TLSv1.2 TLSv1.3; 

    ssl_ciphers 'ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:ECDHE-ECDSA-AES256-SHA384:ECDHE-RSA-AES256-SHA384'; 

    ssl_prefer_server_ciphers on; 

    ssl_session_cache shared:SSL:50m; 

    ssl_stapling on; 

    ssl_stapling_verify on; 

    ssl_dhparam /etc/ssl/certs/dhparam.pem; 

 

    access_log /var/log/nginx/sonarqube.access.log main; 

    error_log  /var/log/nginx/sonarqube.error.log; 

    location / { 

        proxy_set_header Connection ""; 

        proxy_set_header Host $http_host; 

        proxy_set_header X-Real-IP $remote_addr; 

        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; 

        proxy_set_header X-Forwarded-Proto $scheme; 

        proxy_http_version 1.1; 

        proxy_pass http://127.0.0.1:9000; 

    } 

} 

Lưu file  Ctrl+X, then Y. 

Xác nhận Nginx configuration syntax. 

$ sudo nginx -t 

Restart the Nginx service. 

$ sudo systemctl restart nginx 

Secure SonarQube 

Vào the URL https://sonarqube.example.com hay https://IP and log in dùng the username and password admin. 

Nhấp vào tab Quản trị, chọn Bảo mật từ danh sách và nhấp vào tùy chọn thả xuống Người dùng. 

Menu Người dùng SonarQube 

Để cải thiện bảo mật, hãy tạo một người dùng khác để sử dụng cho việc quét mã bằng cách nhấp vào nút Tạo người dùng. 

SonarQube Tạo User Popup 

Nhấp vào nút trong cột Tokens đối với người dùng mới tạo. 

Nhấp vào nút Update Tokens, nhập tên token và nhấp vào nút Generate để tạo token mới cho người dùng. Sao chép và lưu token. 

SonarQube User Token Dialog 

Administration >> Configuration >> Encryption and click the Generate Secret Key button to generate a unique secret key. Copy the saved key to use sau. 

SonarQube Secret Key Page 

$ sudo /opt/sonarqube/conf/sonar.properties 

sonar.secretKeyPath=/opt/sonarqube/conf/sonar-secret.txt 

Tạo the Sonar secret key file. 

$ sudo nano /opt/sonarqube/conf/sonar-secret.txt 

Nhập secret key của bạn đã tạo vào. 

Giới hạn secret file đối với sonarqube user. 

$ sudo chown sonarqube:sonarqube /opt/sonarqube/conf/sonar-secret.txt 

Khởi động lại SonarQube server. 

$ sudo systemctl restart sonarqube 

 Administration >> Configuration >> Encryption section again and fill in your database password. Press the Encrypt button to generate the encrypted password. 

SonarQube Encrypt Properties Page 

Thay thế mật khẩu thực tế trong tệp sonar.properties bằng phiên bản được mã hóa và khởi động lại máy chủ. Lặp lại quy trình với bất kỳ thuộc tính nào khác mà bạn muốn mã hóa. 

13. Install SonarQube's Code Scanner 

Tải the scanner. 

$ wget https://binaries.sonarsource.com/Distribution/sonar-scanner-cli/sonar-scanner-cli-4.7.0.2747-linux.zip 

Giải nén 

 

$ sudo unzip sonar-scanner-cli-4.7.0.2747-linux.zip 

$ sudo mv sonar-scanner-4.7.0.2747-linux /opt/sonarscanner 

$ cd /opt/sonarscanner 

$ sudo nano conf/sonar-scanner.properties 

Tìm line and un-comment line. 

#sonar.host.url=http://localhost:9000 

Thay đổi giá trj url. 

sonar.host.url=https://sonarqube.example.com 

$ sudo chmod +x bin/sonar-scanner 

$ sudo ln -s /opt/sonarscanner/bin/sonar-scanner /usr/local/bin/sonar-scanner
